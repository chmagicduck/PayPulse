# 领域模型

## 1. 事实来源规则

来源：闭环假设

- 页面里的标题、卡片、弹层、列表结构属于 `页面直接事实`
- 跨页重复出现但未显式共享的数据，必须先提升为共享实体，再进入逻辑开发
- 所有无法从静态页直接得到、但又必须存在才能开发的规则，都统一视为 `闭环假设`

## 2. 启动门禁

来源：跨页推理 + 闭环假设

- 应用启动时先读取 `AppBootstrapState`
- `status = needs_setup` 或 key 缺失时，统一进入设置页
- 设置页保存通过后，先持久化 `ProfileSettings`，再执行初始化
- 初始化完成后，将启动状态改为 `ready`，再进入首页
- 数据中心执行全量重置后，必须清除启动状态并重新回到设置页，而不是回个人页

## 3. 初始化流程

来源：闭环假设

初始化在 v1 中一次性完成以下动作：

1. 持久化用户设置
2. 写入 `AppBootstrapState = ready`
3. 初始化头像为默认预设头像
4. 生成当月从 1 日到今日的 `DailyVoyageRecord` 骨架数据
5. 生成默认 `TimeAxisEntry`
6. 初始化 `LabProgress`
7. 清空旧的报表修正映射

默认生成的时间轴事件：

- 生日：`birthday`，分类 `life`，周年模式开启
- 入职纪念日：`career-anniversary`，分类 `career`，周年模式开启
- 退休日：`retirement-day`，分类 `career`，周年模式关闭

## 4. 用户设置的作用

来源：页面直接事实 + 跨页推理

`ProfileSettings` 是全局计算输入源，用于驱动：

- 首页收益和倒计时展示
- 日历里的发薪日、工作日/休息日判断
- 退休倒计时和生命旅程类卡片
- 初始化默认时间轴事件
- 报表的收益换算口径

保存前必须通过最小校验：

- 昵称非空
- 生日早于今天
- 入职日期不早于生日且不晚于今天
- 退休年龄、预期寿命、月薪、发薪日为正数
- 上班时间早于下班时间
- 午休开启时，午休区间必须落在工作区间内

## 5. 每日记录规则

来源：跨页推理 + 闭环假设

`DailyVoyageRecord` 是首页与战报页的共同事实表。

规则：

- 以本地自然日为唯一主键
- 每日记录至少包含：排班工作时长、摸鱼时长、手动修正信息、收益结果和展示状态
- 首页“修正避风时长”修改当天记录
- 战报页“修正时长”修改历史记录
- 两个页面都只能读写同一张事实表

收益换算口径：

- 每月标准工作分钟数 = 当月所有有效工作日的计划工作分钟数之和
- 单位分钟收益 = `monthlySalary / monthlyWorkMinutesOfMonth`
- 每日摸鱼收益 = `moyuDurationMinutes * unitIncomePerMinute`
- 所有正式存储统一使用分或秒，页面展示时再格式化为元和 `HH:mm:ss`

## 6. 日历与排班规则

来源：页面直接事实 + 跨页推理 + 闭环假设

日历的工作日判断由三层叠加：

1. 用户设置的工作模式
2. 当前周是否大周
3. 官方节假日与补班日数据

具体规则：

- `double`：周一到周五工作
- `single-sat`：周一到周六工作
- `single-sun`：周一到周五与周日工作
- `big-small`：基准为双休；大周额外把周六视为工作日，小周保持双休
- 官方 `holiday` 强制视为休息日
- 官方 `makeup` 强制视为工作日
- 发薪日默认取每月 `payDay`；当月不存在该日期时，回落到当月最后一天

日历页“下一关键提醒”选择逻辑：

- 在当前展示月份内收集候选项：下一次发薪日、下一次法定假期起始日、下一次补班日、下一条未来时间轴事件
- 取日期最近的一项作为主提醒
- 若当月不存在候选项，则显示该月摘要文案

## 7. 时间轴规则

来源：页面直接事实 + 跨页推理

`TimeAxisNotebook` 是系统预置分类，不支持用户新增或删除。

`TimeAxisEntry` 规则：

- 标题、日期、分类、是否周年模式为用户可编辑字段
- 周年模式开启时，倒计时基于下一次周年日期
- 周年模式关闭时，未来事件显示倒计时，过去事件显示已过去时长
- 首页的重要日期区块、部分日历提醒都从时间轴派生，不再维护单独的静态列表

## 8. 报表规则

来源：页面直接事实 + 跨页推理 + 闭环假设

`ReportViewState` 是派生视图，不单独持久化。

规则：

- 趋势图默认按最近 30 个自然日生成
- `income` 和 `duration` 两个 tab 共享同一批底层记录
- “近 5 日避风明细”来自最近 5 条 `DailyVoyageRecord`
- 时长占比按日 / 周 / 月 / 年聚合 `moyuDuration` 与 `scheduledWorkDuration`
- “击败全国社畜”在 v1 中不依赖外部真实排行，定义为本地娱乐化评级卡，基于年度摸鱼收益阈值映射展示百分比

## 9. 动力舱规则

来源：页面直接事实 + 跨页推理 + 闭环假设

`LabProgress` 独立持久化，但会读取每日记录或任务日志更新成就进度。

规则：

- 任务模板固定，任务计数可加减，但必须 obey 上下限
- 每日任务在本地 00:00 重置
- 累计积分不随日切清零
- 当前等级由累计积分和静态等级阈值决定

成就进度口径：

- `discipline`：连续工作日完成“准点回港”任务的天数
- `idle-master`：累计摸鱼分钟数
- `whale`：当前自然周“深海补水”任务累计次数

## 10. 数据导出与重置

来源：页面直接事实 + 跨页推理 + 闭环假设

导出规则：

- 导出内容必须覆盖全部核心实体和运行恢复所需元数据
- 导出格式为单个 JSON 包
- v1 只做本地导出，不做云端同步

重置规则：

- 重置清除用户设置、头像、启动状态、每日记录、时间轴事件、动力舱进度、报表修正映射和展示偏好
- 重置成功后立即回到未初始化状态
- 重置后首次进入任何业务页，都应被门禁导回设置页
