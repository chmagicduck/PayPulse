# v1.0.0 Implementation Gap Log

## 1. 动力舱成就历史精度

- 模块：`lab`
- 现象：`discipline`（连续 7 天准点下班）与 `whale`（一周喝水 40 次）当前可以更新进度，但不是基于独立的历史任务日志精确重算。
- 阻塞原因：现有 PRD 与数据模型没有单独定义“按天任务日志 / 连续打卡历史 / 周维度计数历史”实体，当前轮次优先落地核心业务闭环。
- 临时处理：使用当前任务状态、累计积分和每日记录摘要做近似进度更新，并保留日切 / 周切重置。
- 后续建议：新增 `LabTaskLog` 或 `DailyTaskSnapshot`，让连续成就和周成就都从历史表精确派生。
- 是否影响发布：否

## 2. 年度统计样本深度

- 模块：`report`
- 现象：年度卡片和趋势图已经基于正式 `DailyVoyageRecord` 派生，但初始化默认只生成“当月 1 日到今日”的记录，因此年度统计的样本深度取决于真实使用天数。
- 阻塞原因：PRD 明确初始化只生成当月骨架数据，本轮不额外虚构全年历史事实。
- 临时处理：所有报表统计坚持使用真实本地记录，不再读取静态 mock。
- 后续建议：若需要更完整年度视图，可新增按月回填策略或更长时间的初始化种子策略。
- 是否影响发布：否

## 3. 日历年份范围

- 模块：`calendar`
- 现象：正式日历规则当前只保证 2026 年完整可用。
- 阻塞原因：本轮实现遵循 v1 假设，只完成 2026 年法定假期、补班和排班叠加逻辑。
- 临时处理：页面完整支持 2026 年 12 个月；其他年份不在当前版本承诺范围。
- 后续建议：将节假日数据源抽成可扩展年份表，再把 month range 扩展为跨年选择器。
- 是否影响发布：否
