# 战报页 Ocean Report

## 1. 页面目标

来源：页面直接事实 + 跨页推理

- 用统一的每日记录展示趋势、年度统计、时长占比和近 5 日明细
- 提供历史记录修正入口

## 2. 入口与前置条件

来源：跨页推理

- 入口：tab 战报页
- 前置条件：已完成初始化
- 依赖：`DailyVoyageRecord`、`ReportViewState`

## 3. 信息结构

来源：页面直接事实

- 收益趋势图
- 年度航行成就卡片
- 时长占比统计
- 近 5 日避风明细
- 修正时长弹层

## 4. 核心交互

来源：页面直接事实 + 跨页推理

- 收益 / 时长 tab 切换同一批趋势数据的不同维度
- 日 / 周 / 月 / 年切换只变更聚合范围
- 点击图表柱子显示当前柱状值提示
- 点击历史卡片打开修正弹层，保存后写回指定日期的 `DailyVoyageRecord`

## 5. 页面状态

来源：页面直接事实

- `activeTab`
- `timeRange`
- `currentRatio`
- `ratioRingSrc`
- `historyItems`
- `showEditModal`
- `editModalVisible`
- `editingIndex`
- `editH / editM / editS`
- `pressStates`
- `iconAnimations`

## 6. 依赖的共享数据 / 共享规则

来源：跨页推理

- 趋势图来自最近 30 天 `DailyVoyageRecord`
- 年度卡片为派生统计，不单独持久化
- 历史修正和首页当天修正共用同一事实表

## 7. 空态 / 异常态 / 边界情况

来源：闭环假设

- 当历史记录不足 5 天时，按实际记录条数展示
- 聚合分母为 0 时，占比统一显示 0%
- 若某天没有摸鱼时长，趋势图该天显示 0 值而不是缺口

## 8. 验收清单

- 首页与战报修正互相可见
- 切换 `income` / `duration` 不会改变底层记录
- 切换时间范围时占比逻辑正确
- 历史记录来自每日事实表而不是战报页私有存储
