# 首页 Dashboard

## 1. 页面目标

来源：页面直接事实 + 跨页推理

- 展示当日摸鱼收益、离岗倒计时和收益占比
- 预览今日任务进度
- 展示常规时间提醒、重要时间轴事件和人生旅程看板
- 提供当天摸鱼时长修正入口

## 2. 入口与前置条件

来源：跨页推理

- 入口：tab 首页、设置页初始化完成后的默认落地页
- 前置条件：`AppBootstrapState.status = ready`
- 依赖：当日 `DailyVoyageRecord`、`ProfileSettings`、`TimeAxisEntry`、`LabProgress` 摘要

## 3. 信息结构

来源：页面直接事实

- 顶部品牌区
- 收益主卡
- 今日任务预览
- 常规波动卡片
- 岁月坐标卡片列表
- 人生航程看板
- 修正时长弹层

## 4. 核心交互

来源：页面直接事实 + 跨页推理

- 点击眼睛图标切换金额显隐，持久化到 `home.amount.visibility`
- 点击“避风中”切换按钮只改变当天展示状态，真实时长以手动修正结果为准
- 点击设置按钮打开修正弹层，保存后写回当天 `DailyVoyageRecord`
- 点击常规波动和岁月坐标卡片只做反馈或跳转准备，不单独产生新事实表
- 点击人生航程卡片切换时间展示模式，模式仅为页面局部状态

来源：页面直接事实

- `iconAnimations`

## 6. 依赖的共享数据 / 共享规则

来源：跨页推理

- 收益、时长、占比来自当天 `DailyVoyageRecord`
- 常规波动来自 `ProfileSettings` + 日历规则
- 岁月坐标来自 `TimeAxisEntry` 的最近 3 条可展示事件
- 人生航程来自 `ProfileSettings` 派生

## 7. 空态 / 异常态 / 边界情况

来源：闭环假设

- 当天记录不存在时，进入页面前自动创建零值记录
- 时间轴可展示事件不足 3 条时，只显示实际条数，不补假数据
- 修正时长输入必须被限制在合法时间范围内

## 8. 验收清单

- 修改当天时长后，首页收益、占比和战报数据同步更新
- 金额显隐状态刷新页面后保持一致
- 岁月坐标内容来自时间轴而不是首页私有静态列表
- 人生航程卡片切换仅影响展示，不写入业务持久化
