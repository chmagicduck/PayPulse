# 加入社区

| 项 | 内容 |
| --- | --- |
| route | `features/community/pages/join` |
| prototype | `docs/prototype/1.0.0/features.community.pages.join.ts` |
| 主要入口 | 个人基地 |

## 页面目标

以单页方式介绍官方社群价值，并承接用户加入社群的核心转化动作。

## 入口与前置条件

| 来源 | 条件 |
| --- | --- |
| 个人基地“加入社区” | 无 |

## 信息结构

| 顺序 | 区块 | 说明 |
| --- | --- | --- |
| 1 | 顶部导航 | 返回、标题和分享入口。 |
| 2 | 英雄头图区 | 展示官方社群定位、口号和统计数据。 |
| 3 | 加群主卡 | 主 CTA 与加入说明。 |
| 4 | 权益说明区 | 展示社群权益卡片。 |

## 核心交互

| 动作 | 结果 |
| --- | --- |
| 点击立即加入 | 进入官方加群引导流程。 |
| 点击分享 | 调起平台分享能力或展示占位反馈。 |

## 页面状态

| 状态 | 类型 | 说明 |
| --- | --- | --- |
| `shareStatus` | enum | 分享反馈占位。 |

## 依赖的共享数据 / 共享规则

- 读取固定展示 schema，包括社群统计、权益卡和加入入口状态。
- 使用企业微信插件 `materialPlugin` 承接官方群入口。
- 官方入口和外链边界见 [domain-model.md](../domain-model.md)。

## 空态 / 异常态 / 边界情况

- 若当前加群入口不可用，主 CTA 应降级为“查看加入说明”而非不可解释的失效按钮。
- 分享能力不可用时，保留按钮但给出明确提示。
- 页面统计数据缺失时，仍应保留基础文案与主 CTA。
- `app.json` 需提前注册企业微信插件：
  - `plugins.materialPlugin.version = "1.0.13"`
  - `plugins.materialPlugin.provider = "wx4d2deeab3aed6e5a"`

## 验收清单

- [ ] 用户能清楚理解社群价值。
- [ ] 主 CTA 存在明确的加入承载方式或降级方案。
- [ ] 异常情况下仍能提供可执行的说明路径。
- [ ] 页面从个人基地进入路径稳定。
