# 个人基地 Profile Home

## 1. 页面目标

来源：页面直接事实 + 跨页推理

- 汇总用户身份信息和累计摸鱼打卡数据
- 作为设置、时间轴、日历、社区、关于和数据中心的入口页
- 提供头像选择弹层

## 2. 入口与前置条件

来源：页面直接事实 + 跨页推理

- 入口：tab 个人基地
- 前置条件：已完成初始化
- 依赖：`ProfileSettings`、头像、累计打卡摘要

## 3. 信息结构

来源：页面直接事实

- 顶部品牌区
- 个人身份卡
- 航行控制台入口
- 关于与社区入口
- 安全与存储入口
- 头像选择弹层

## 4. 核心交互

来源：页面直接事实 + 跨页推理

- 点击头像打开预设头像弹层，确认后保存头像
- 点击控制台卡片跳转到设置、时间轴、日历
- 点击支持类卡片跳转到社区、关于
- 点击存储卡片跳转到数据中心

## 5. 页面状态

来源：页面直接事实

- `showAvatarModal`
- `avatarModalVisible`
- `currentAvatar`
- `draftAvatar`
- `pressStates`
- `iconAnimations`

## 6. 依赖的共享数据 / 共享规则

来源：跨页推理

- 昵称来自 `ProfileSettings.nickname`
- 累计摸鱼打卡天数来自 `DailyVoyageRecord`
- 头像持久化到 `profile.avatar`

## 7. 空态 / 异常态 / 边界情况

来源：闭环假设

- 头像为空时使用默认预设头像
- 已删除或损坏的头像资源回退到默认头像
- 累计打卡天数在无记录时显示 0 天

## 8. 验收清单

- 头像修改刷新后保持一致
- 各入口跳转链与 `page-map.md` 一致
- 用户名称不再从页面静态 mock 读取
