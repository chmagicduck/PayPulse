# 关于软件

| 项 | 内容 |
| --- | --- |
| route | `features/about/pages/about` |
| prototype | `docs/prototype/1.0.0/features.about.pages.about.ts` |
| 主要入口 | 个人基地 |

## 页面目标

向用户清晰介绍产品品牌、当前版本、更新历史和设计理念，增强信任感与产品理解。

## 入口与前置条件

| 来源 | 条件 |
| --- | --- |
| 个人基地“关于软件” | 无 |

## 信息结构

| 顺序 | 区块 | 说明 |
| --- | --- | --- |
| 1 | 顶部导航 | 返回与标题。 |
| 2 | 品牌头图区 | Logo、产品名、口号、版本和能力标签。 |
| 3 | 标签切换区 | 在更新日志与设计理念之间切换。 |
| 4 | 内容区 | 更新日志时间线或理念卡片。 |
| 5 | 页尾版权区 | 展示版权和品牌归属。 |

## 核心交互

| 动作 | 结果 |
| --- | --- |
| 点击“更新日志” | 展示版本时间线。 |
| 点击“设计理念” | 展示理念卡片组。 |

## 页面状态

| 状态 | 类型 | 说明 |
| --- | --- | --- |
| `activeTab` | enum | `changelog/concept` 双标签切换。 |

## 依赖的共享数据 / 共享规则

- 读取固定 schema，包括当前版本、更新日志和理念卡片。
- 版本号与更新日志首条必须保持一致。
- 品牌展示规则见 [domain-model.md](../domain-model.md)。

## 空态 / 异常态 / 边界情况

- 若更新日志仅有当前版本，也要完整展示版本、日期和摘要。
- 若某一标签下暂无内容，需要展示占位说明而不是空白区域。
- 版本号更新时，头图区和更新日志首条需保持一致。

## 验收清单

- [ ] 头图区、标签切换和内容区都可正常展示。
- [ ] 两个标签切换流畅且内容正确。
- [ ] 当前版本号与日志首条一致。
- [ ] 空内容和单版本场景已处理。
