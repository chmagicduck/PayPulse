# 动力舱 Lab

## 1. 页面目标

来源：页面直接事实 + 跨页推理

- 承载积分、等级、日常任务和成就进度
- 让用户以轻量互动方式维护日常任务完成情况

## 2. 入口与前置条件

来源：跨页推理

- 入口：tab 动力舱
- 前置条件：已完成初始化
- 依赖：`LabProgress`、部分 `DailyVoyageRecord`

## 3. 信息结构

来源：页面直接事实

- 当前等级卡
- 等级轴
- 日常巡航任务列表
- 荣誉勋章列表

## 4. 核心交互

来源：页面直接事实 + 跨页推理

- 点击等级轴切换当前查看等级，不改业务事实
- 点击任务加减按钮修改对应任务计数
- 任务计数变化后，联动更新今日积分、累计积分和当前等级
- 成就进度按规则自动派生，不提供手动编辑入口

## 5. 页面状态

来源：页面直接事实

- `totalHappiness`
- `todayHappiness`
- `rankIndex`
- `pressStates.rankIndex`
- `pressStates.taskActionKey`
- `iconAnimations.header`
- `iconAnimations.activeRankIndex`
- `iconAnimations.activeTaskId`

## 6. 依赖的共享数据 / 共享规则

来源：跨页推理

- 等级阈值与任务模板来自静态配置
- 日切重置逻辑依赖 `LabProgress.lastDailyResetDate`
- 成就进度会读取任务日志和部分每日记录结果

## 7. 空态 / 异常态 / 边界情况

来源：闭环假设

- 任务计数不能低于 0，也不能高于各自 `limit`
- 跨过本地 00:00 时，日任务计数与今日积分重置，累计积分不清零
- 已完成成就再次满足条件时，不重复发放奖励积分

## 8. 验收清单

- 任务计数 obey 上下限
- 日切后只重置当日维度数据
- 累计积分变化会推动等级变化
- 成就进度与任务 / 每日记录规则一致
