# 档案设置页 Profile Settings

## 1. 页面目标

来源：页面直接事实 + 跨页推理

- 收集全局计算所需的用户档案与排班参数
- 在首次启动和重置后承担初始化入口

## 2. 入口与前置条件

来源：跨页推理

- 入口：首次启动强制进入、重置后强制进入、个人页手动进入
- 前置条件：无；未初始化时是全局唯一允许进入的业务页

## 3. 信息结构

来源：页面直接事实

- 顶部导航
- 说明 Banner
- 船员身份识别
- 航程工龄坐标
- 航行排班计划
- 保存按钮

## 4. 核心交互

来源：页面直接事实 + 跨页推理

- 输入昵称、生日、入职时间、退休年龄、预期寿命、月薪、发薪日
- 选择性别、工作模式、是否大周
- 选择上下班时间和午休区间
- 点击保存时执行校验；若当前未初始化，则保存成功后触发初始化

## 5. 页面状态

来源：页面直接事实

- `form`
- `pressStates.save`
- `iconAnimations.banner`

## 6. 依赖的共享数据 / 共享规则

来源：跨页推理

- 页面输入写入 `ProfileSettings`
- 保存成功时触发 `AppBootstrapState` 和初始化流程
- 工作模式和时间配置将影响首页、战报、日历和人生旅程

## 7. 空态 / 异常态 / 边界情况

来源：闭环假设

- 未通过校验时不允许保存
- 未初始化用户点击返回时，仍需被门禁重新导回本页
- 开启午休但未填写完整午休时间时，保存失败

## 8. 验收清单

- 首次进入应用时首先看到本页
- 保存有效设置后创建初始化数据并进入首页
- 重置后重新回到本页
- 修改设置后，依赖页面的派生结果更新
