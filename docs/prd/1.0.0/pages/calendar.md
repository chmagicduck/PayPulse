# 日历页 Calendar

## 1. 页面目标

来源：页面直接事实 + 跨页推理

- 展示当前月份的工作日、休息日、节假日、补班和发薪日
- 展示选中日期的说明与本月下一关键提醒

## 2. 入口与前置条件

来源：页面直接事实 + 跨页推理

- 入口：个人基地入口、首页“查看日历”入口
- 前置条件：已完成初始化
- 依赖：`ProfileSettings`、节假日规则、`TimeAxisEntry`

## 3. 信息结构

来源：页面直接事实

- 顶部导航
- 月份切换头部
- 月历网格
- 图例
- 当日状态详情卡
- 下一关键提醒卡

## 4. 核心交互

来源：页面直接事实 + 跨页推理

- 点击上月 / 下月切换当前展示月份
- 点击日期切换当前选中日
- 当前选中日驱动详情卡文案变化
- 本月提醒卡由共享日历规则派生

## 5. 页面状态

来源：页面直接事实

- `monthIndex`
- `selectedDay`
- `currentMonth`
- `currentDetail`
- `selectedStatus`
- `emptySlots`

## 6. 依赖的共享数据 / 共享规则

来源：跨页推理

- 工作日判断依赖设置中的工作模式和官方节假日数据
- 发薪日由 `ProfileSettings.payDay` 派生
- 提醒卡可读取时间轴未来事件

## 7. 空态 / 异常态 / 边界情况

来源：闭环假设

- 当前月份无官方节假日数据时，仍需按工作模式和发薪日生成基础日历
- 当月不存在配置发薪日时，回落到该月最后一天
- 月份切换范围至少覆盖当前运行月份及相邻月份

## 8. 验收清单

- 修改工作模式或发薪日后，日历结果同步更新
- 补班和法定节假日能覆盖普通工作模式结果
- 日期详情与图例状态一致
