# 验收清单

## 版本主链路

来源：跨页推理 + 闭环假设

1. 首次安装启动
2. 被门禁导入设置页
3. 保存有效设置
4. 完成初始化并进入首页
5. 首页、战报、动力舱、个人页可正常访问
6. 时间轴可新增/编辑/删除事件
7. 日历和首页相关提醒随设置或时间轴变化而变化
8. 数据中心可导出和重置
9. 重置后重新回到设置页

## 核心验收项

- 首次启动且无 `app.bootstrap.state` 时，不能直接停留在首页
- 设置页保存后，必须写入设置、启动状态和默认业务数据
- 首页修正当天时长后，战报页对应日期同步变化
- 战报页修正历史时长后，首页当天以外数据不受影响
- 时间轴编辑后，首页岁月坐标和日历提醒按新数据变化
- 修改发薪日、工作模式、午休配置后，日历和收益口径按新配置生效
- 动力舱任务计数 obey 上下限，00:00 后执行日切重置
- 导出包覆盖全部核心实体
- 重置后必须清掉启动状态，重新进入设置页流程

## 回归重点

- 首页与战报是否仍共用同一张每日记录事实表
- 时间轴是否仍是首页重要日期的唯一事实源
- 页面文档是否没有重复维护共享规则
- storage key 是否与 `data-model.md` 一致
