# 版本验收

## 结构验收

- [ ] `docs/prd/1.0.0/` 目录完整存在，且包含入口、共享模型、页面文档和版本验收文档。
- [ ] 页面文档数量与 `docs/prototype/1.0.0/` 原型文件数量一致，均为 10 页。
- [ ] 每个页面文档都能明确映射到一个 route 和一个 prototype 文件。
- [ ] 仓库内不再保留 `docs/prd/feature-*` 旧 PRD。
- [ ] 仓库内不再保留旧的单文件 PRD 入口。

## 内容验收

- [ ] `overview.md` 能说明版本目标、范围和非目标。
- [ ] `domain-model.md` 能独立覆盖核心业务规则，不依赖页面文档补全。
- [ ] `data-model.md` 能独立覆盖共享状态、存储 key、默认值和关键约束。
- [ ] `pages/*.md` 只保留页面信息，不再重复整份共享 schema。
- [ ] `acceptance.md` 覆盖跨页面链路，而不是简单拼接页面 checklist。

## 跨页面链路验收

### 档案主链路

- [ ] 用户能从个人基地进入航行档案设置。
- [ ] 保存档案后，首页收益、日历发薪日和战报聚合都能共享同一份输入口径。
- [ ] 档案缺省时，首页和日历都有默认占位和降级说明。

### 日期主链路

- [ ] 用户能在岁月坐标设置里新增、编辑、删除条目。
- [ ] 首页只展示预览，日历能消费同一批日期事实源。
- [ ] 周年提醒和普通倒计时文案口径一致。

### 成长主链路

- [ ] 动力室完成任务后，快乐值和等级摘要立即变化。
- [ ] 个人基地读取到的等级摘要与动力室一致。
- [ ] 首页任务预览只展示摘要，不复制动力室全量数据。

### 数据治理主链路

- [ ] 数据管理中心能够导出正式业务 key 的 JSON 备份文本。
- [ ] 数据重置必须经过确认，且执行结果与提示一致。
- [ ] 云端同步入口保持占位，不误导为正式能力。

## 风险与回归重点

- [ ] 不存在同一业务规则在多个页面写出不同口径的情况。
- [ ] 不存在共享数据结构散落在页面文档重复定义的情况。
- [ ] 不存在把占位能力写成正式需求的文案误导。
- [ ] 不存在删除旧 PRD 后仍有仓库说明引用旧路径的情况。
