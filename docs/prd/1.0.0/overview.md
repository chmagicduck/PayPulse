# 1.0.0 版本概览

## 版本目标

1.0.0 是 PayPulse 的首个完整版本说明，目标是把“档案配置 -> 日常计时 -> 趋势复盘 -> 日期管理 -> 激励反馈 -> 数据治理”串成一条可实现、可测试、可维护的本地优先体验链路。

本版重点解决：

- 用户首次进入后，有明确的档案录入入口和默认首页
- 首页能基于档案、任务、日期等共享数据形成日常总览
- 用户能维护关键日期、查看日历节奏、复盘收益与时长趋势
- 用户能通过动力室获得任务反馈和等级成长感
- 用户能在个人基地统一进入工具页、品牌页和数据治理页
- 所有核心数据都具备清晰的本地存储口径与重置边界

## 本版范围

### 正式交付

- 航行档案设置
- 首页日常总览与当日计时
- 岁月坐标管理
- 月视图日历
- 洋流战报趋势与近 7 日修正
- 动力室任务与等级摘要
- 个人基地入口中枢
- 数据导出与本地重置
- 加入社区页
- 关于软件页

### 非目标

- 真实云端同步
- 多端账号体系
- 农历录入与换算
- 多主题皮肤系统
- 下载文件式备份
- 战报高级筛选、分享海报和复杂排行

## 页面覆盖

| 分组 | 页面 |
| --- | --- |
| 主路径 | 首页、洋流战报、动力室、个人基地 |
| 配置与工具 | 航行档案设置、岁月坐标设置、日历、数据管理中心 |
| 品牌与增长 | 加入社区、关于软件 |

## 版本边界

- 1.0.0 的事实源只来自 `docs/prototype/1.0.0/` 与本目录 PRD。
- 页面实现允许做静态展示细节优化，但不得新增未在本目录登记的业务能力。
- 共享规则必须从页面文档中抽离，避免“同一规则在多个页面各写一版”。
- 正式能力和占位能力必须在文档中显式区分，避免误导实现与测试。

## 关键依赖链

1. `profile.settings` 是首页、日历、战报、岁月坐标的基础输入。
2. `lab.progress` 同时服务动力室本页和个人基地等级摘要。
3. `time.axis.entries` 同时服务岁月坐标本页、首页预览和日历关键日期展示。
4. `home.daily.session` 为首页实时态提供事实源，并参与战报聚合。
5. 数据管理中心对上述业务 key 提供统一导出和重置能力。
