# v1.0.0 概览

## 版本目标

来源：跨页推理

- 建立一套以用户设置为输入源、以每日记录为核心事实表的本地业务闭环
- 让首页、报表、时间轴、日历、动力舱和数据管理共用同一批核心实体
- 用版本 PRD 替代“从静态页直接猜逻辑”的开发方式

## 本期范围

来源：跨页推理

- 启动门禁：首次进入和清空数据后，必须先进入设置页
- 用户设置：保存全局计算输入，并作为初始化触发点
- 每日记录：沉淀首页与报表共用的时间和收益事实
- 时间轴：管理用户重要事件并驱动首页/日历的相关提醒
- 动力舱：维护任务、积分、等级和成就进度
- 数据中心：支持本地导出与全量重置

## 本期非目标

来源：闭环假设

- 登录、账号体系、云同步、多设备合并
- 真实全国排行榜或外部统计服务
- 实时在线协作、社群打通、远程配置下发
- 可编辑的系统分类、复杂多用户权限

## 版本原则

来源：闭环假设

- 本地数据优先，所有核心能力必须能在离线场景工作
- 静态 mock 只用于识别字段和页面关系，不直接进入正式逻辑
- 共享规则只维护一份，页面文档只写页面局部行为
- 所有核心流程都要有启动、空态、重置和回归口径
